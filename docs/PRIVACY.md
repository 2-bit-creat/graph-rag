# 개인정보 처리 플레이북

> **이 문서의 목적**: 이 프로젝트에서 개인정보에 손대는 모든 작업(새 LLM 호출 추가, 새 데이터 테이블, 새 에이전트/외부 서비스 연동, 새 화면 등) 전에 먼저 읽는 문서입니다. 데이터가 어디로 흐르는지, 법적으로 무엇이 필요한지, 이미 무엇이 구현되어 있는지, 무엇이 의도적으로 보류되었는지를 한 곳에 모아 매번 처음부터 다시 조사하지 않도록 합니다.
>
> 마지막 갱신: 2026-07-12 (P0/P1/P2 컨설팅 완료 시점). **새 기능을 추가하면 이 문서도 같이 갱신하세요** — 특히 "데이터 흐름 지도"와 "신규 기능 체크리스트" 섹션.

## 1. 이 앱이 다루는 개인정보의 성격

- 개인 일기·음성·대화 기록 + 이로부터 생성되는 지식그래프(인물·개념·진술 노드)
- 일기 특성상 건강·신념 등 민감정보가 자연히 유입될 수 있음
- 화자 식별 기능은 음성에서 **생체인식 특징정보(성문)**를 추출 — 한국 개인정보보호법상 별도 동의가 필요한 민감정보
- 계정은 비밀번호 없는 핸들 방식(무비번, 의도적 설계 — `POST /auth/simple`, `backend/app/routers/auth.py`)

## 2. 데이터 흐름 지도

**새 기능이 아래 표에 없는 목적지로 데이터를 보내면, 반드시 이 표를 갱신하고 §4·§6을 재검토하세요.**

| 수신자 | 전송 내용 | 코드 위치 | 국외이전 근거 |
|---|---|---|---|
| OpenAI (gpt-4o-mini) | 일기 원문, 화자명, 인물/별칭 힌트, 채팅 전문+히스토리+요약, 그래프 트리플, 퀴즈 시드 | `kg_build.py`, `graph_chat.py`, `chat_distill.py`, `quiz_bundle.py` 등 | 처리방침 §3 명시, 처리위탁 구성 |
| OpenAI (whisper-1) | 일기 음성 원본 파일 | `journal_pipeline.py` | 상동 |
| OpenAI (embeddings) | Statement 원문 + 인물 이름/별칭 개별 임베딩 | `rag.py`, `crud.py` (`index_identity_alias`) | 상동 |
| Deepgram (옵션, 기본 OFF) | 음성 원본 WAV | `speaker_diarization.py` | 상동 |
| Microsoft Edge TTS | 퀴즈 문장 (일기 파생) | `quiz_audio_engine.py` | 상동. **비공식 무료 API — 출시 전 교체 권장**(§7) |
| AWS S3 (옵션) | 음성 원본 | `storage.py` | 처리방침에 인프라로 명시 필요 시 추가 |

마스킹/가명처리 레이어는 **없음**(의도적 보류, §7 참고). 애널리틱스·크래시리포팅은 전혀 사용하지 않음(양호, 유지할 것).

### 저장 위치 결정
지식그래프·일기 원문은 **서버(Postgres+pgvector) 저장을 유지**하기로 결정함(2026-07-11). 이유: LLM 호출이 어차피 서버를 경유해 데이터가 외부로 나가는 구조라 온디바이스 이전의 프라이버시 이득이 제한적이고, pgvector 검색·화자분리 파이프라인을 클라이언트에서 재구현하는 비용이 과대. 신규 기능이 이 전제를 깨는 경우(예: 완전 오프라인 모드) 재검토 필요.

## 3. 한국 법규 요건 (요약)

### 개인정보보호법 (PIPA)
- **국외이전(제28조의8)**: 위 표의 각 목적지는 국외 처리위탁. 개인정보처리방침에 이전 항목·국가·목적·보유기간을 고지하면 별도 동의 없이 적법(현재 구현: `/legal/privacy-policy`, §5 참고). **새 국외 서비스를 추가할 때마다 처리방침에 행을 추가할 것.**
- **생체인식 특징정보(시행령)**: 화자식별 음성 특징정보(성문)는 민감정보 → **별도 동의 필수**. 원본은 특징정보 생성 후 파기가 원칙이나, 본 프로젝트는 "음성 재생 유지" 결정에 따라 **원본을 보관**하는 대신 성문 생성 자체를 동의로 게이팅함(§5).
- **안전성 확보조치**: 전송구간 암호화(TLS), 접근통제, 최소보존.
- **파기(제21조)**: 목적 달성 시 지체없이 파기. 디버그 로그·회원탈퇴 처리에 반영됨(§5).
- **정보주체 권리**: 열람(내보내기)·정정·삭제·동의철회.
- 만 14세 미만은 법정대리인 동의 필요 — **미구현**(연령확인 절차 없음, §7 참고).

### AI기본법 (2026-01-22 시행)
- 생성형 AI 기반 서비스임을 **사전 고지**(구현: `/legal/ai-disclosure`)
- AI 생성물에 **표시**(구현: 채팅 응답에 "AI 생성" 라벨. 퀴즈·정제문에는 미적용 — §7)
- 과태료 계도기간 ~2027 예상이나, 고지·표시 의무 자체는 시행 중

## 4. LLM 전송 마스킹 정책 (결정됨)

완전 익명화는 하지 않기로 결정(2026-07-11). 이유:
1. 음성 STT·원문 정제는 구조적으로 원문 없이 불가능
2. 국외이전 동의(처리방침 고지)로 이미 적법
3. 가명화 레이어(Identity alias→토큰 치환)는 **검토 후 보류**(2026-07-12) — LLM 추출·채팅 품질 저하 리스크 대비 ROI가 낮다고 판단. 필요 시 `_existing_nodes_hint`(kg_build.py), graph_chat 컨텍스트, 퀴즈 시드에 적용 가능하도록 설계는 남겨둠.

**새 LLM 호출을 추가할 때**: 원문에 실명·민감정보가 포함되는지 확인하고, 포함된다면 이 정책(마스킹 안 함, 국외이전 고지로 커버)이 여전히 적절한지 재검토. 새로운 국외 서비스라면 §2 표와 처리방침에 추가.

## 5. 구현 현황 (완료)

### P0 — 출시 전 필수 보안 (커밋 `861c9ce`, `0359f44`, `211ffa2`)
- **인증 격리**: 무토큰 요청은 프로덕션에서 401(`deps.py get_request_user`), 개발 환경만 dev 유저 폴백. `graph.py`/`graph_chat.py` 전 엔드포인트가 `dev_user_dep` 대신 `request_user_dep`로 계정별 격리. 노드·엣지 CRUD에 소유권 검증(`crud.py`).
- **환경 게이팅**: `config.py`의 `environment`("development"|"production") 플래그가 모든 프로덕션 하드닝의 스위치. `is_production`, `jwt_secret_is_insecure`, `db_credentials_are_insecure`, `debug_enabled` 프로퍼티.
- **전송 보안**: 프로덕션 CORS는 화이트리스트만(LAN/localhost 정규식 제외), Android 릴리스 빌드는 cleartext 차단(`src/debug`만 평문 허용), AWS 템플릿에 `Environment`/`JwtSecret` 파라미터 배선.
- **보존 최소화**: 디버그 추적(`pipeline_trace.py`)이 프로덕션에서 원문 프롬프트·음성을 디스크/DB에 남기지 않고 구조만 유지(`_redact_trace`). `debug_runs/`는 `DEBUG_RUNS_RETENTION_DAYS`(기본 7일)로 자동 정리. `/kg/debug/runs`·trace 엔드포인트는 프로덕션 404.
- **완전 파기**: `DELETE /auth/me`가 DB 캐스케이드 + `uploads/{user_id}/`(음성·단어장·표현 파일) + `debug_runs/{entry_id}/` + S3 프리픽스까지 삭제(`storage.py purge_user_storage`).

### P1 — 법적 컴플라이언스 (커밋 `dc99f5d`, `9fdb068`, `5b73ca1`)
- **동의 추적**: `User.consent_version`/`consented_at`/`speaker_id_consent_at`(성문 별도 동의). `POST /auth/consent`.
- **온보딩 동의 게이트**: `mobile/lib/screens/consent_screen.dart` — 필수(개인정보+국외이전) + 선택(화자식별) + AI기본법 고지. 미동의 계정은 홈 진입 불가(`account_controller.dart needsConsent`).
- **처리방침**: `backend/app/legal/privacy_policy_ko.md`(초안, `{{플레이스홀더}}` 포함) + `GET /legal/privacy-policy`(공개) + 앱 내 `privacy_policy_screen.dart`.
- **데이터 내보내기**: `GET /auth/me/export`(`data_export.py`, 비밀번호·벡터 컬럼 제외 전체 JSON) + 설정 화면의 "내 데이터 내보내기"(파일 저장, 미지원 시 클립보드 폴백).
- **성문 동의 게이팅**: `speaker_profiles.py process_entry_speaker_profiles`가 `speaker_id_consent_at` 없으면 voiceprint 생성을 스킵(음성 원본은 재생 위해 유지 — §2 저장 위치 결정과 일치).
- **AI 생성 표시**: `graph_chat_panel.dart`의 어시스턴트 버블에 "AI 생성" 라벨.

### P2 — 저장 보안 (커밋 `7785936`, `d507118`)
- **모바일 토큰**: `shared_preferences` 평문 → `flutter_secure_storage`(Keychain/Keystore), 레거시 자동 마이그레이션.
- **DB 크리덴셜 가드**: 프로덕션에서 기본 `graphrag:graphrag` 크리덴셜이면 기동 거부(`main.py`).
- **docker-compose**: Postgres/MinIO 크리덴셜 env 파라미터화(개발 기본값 유지).
- **암호화·TTS**: 컬럼 암호화는 pgvector 검색과 충돌하므로 **미채택** — 대신 관리형 DB(RDS/Supabase) 디스크 암호화를 README에 권장 명시. Edge TTS 교체(Azure/flutter_tts)도 README에 권장 명시(코드 미변경, 배포 시 결정).

## 6. 신규 기능 추가 시 체크리스트

새 기능(특히 LLM 호출, 새 데이터 저장, 외부 서비스/에이전트 연동, 새 화면)을 만들 때마다 확인:

1. **개인정보/민감정보가 새 목적지로 나가는가?** → §2 표에 추가, 처리방침(`privacy_policy_ko.md`) §3 표에 국외이전 항목 추가.
2. **원문 프롬프트/응답을 어딘가에 저장하는가?** → `pipeline_trace.py`의 `debug_enabled`/`_redact_trace` 패턴을 재사용할 것. 무기한 보존 금지.
3. **새 DB 테이블/컬럼에 사용자 데이터가 들어가는가?** →
   - `user_id` FK로 소유권 명확히 할 것 (crud 함수에 `owner_id`/`user_id` 검증 패턴 — `crud.py`의 `delete_node`/`create_edge` 참고)
   - `data_export.py export_user_data`에 포함할지 검토(열람권 대응)
   - `storage.py purge_user_storage` 또는 `auth.py delete_me`의 캐스케이드에 파기 로직 포함할지 검토
4. **생체정보(음성 특징, 얼굴 등)를 다루는가?** → 반드시 별도 동의 필드+게이팅 필요(성문 패턴을 참고: `User.speaker_id_consent_at` + `speaker_profiles.py`의 조기 반환 가드).
5. **AI가 생성한 콘텐츠를 사용자에게 보여주는가?** → "AI 생성" 표시 필요(AI기본법). 현재 채팅에만 있음 — 퀴즈·정제문에 새 표면이 생기면 동일하게 적용.
6. **새 에이전트/외부 API를 연동하는가?** → 그 서비스가 사용자 콘텐츠를 받는다면 §2 표 + 처리방침에 추가하고, 해당 서비스의 데이터 보존 정책(학습 사용 여부, 보존기간)을 확인해 처리방침에 반영.
7. **인증이 필요한 새 엔드포인트인가?** → `request_user_dep` 사용(무토큰 시 프로덕션 401, 개발만 폴백). `dev_user_dep`는 죽은 코드이므로 사용 금지.
8. **불특정 다수가 접근 가능한 새 디버그/관리자 엔드포인트인가?** → `deps.require_debug_enabled` 패턴으로 프로덕션 차단할 것.

## 7. 의도적으로 보류/미구현된 항목

작업 중 "이거 왜 안 되어 있지?"라고 생각할 수 있는 것들 — 전부 **검토 후 의도적으로 보류**된 것이며, 재작업 전에 아래 이유를 먼저 확인할 것.

| 항목 | 상태 | 이유 |
|---|---|---|
| 핸들 계정 비밀번호/디바이스키 | 미구현 (현행 유지) | 무비번 설계를 뒤집는 제품 결정이라 사용자가 보류 선택. 계정 격리(P0)는 되어 있어 핸들만 알아도 타 계정 데이터는 못 봄(단, 로그인 자체는 여전히 핸들만으로 가능) |
| Identity alias 가명화(토큰 치환 레이어) | 미구현 (보류) | 국외이전 동의로 이미 적법, LLM 품질 리스크 대비 ROI 낮다고 판단(2026-07-12). 설계는 §4에 남겨둠 |
| DB 컬럼 레벨 암호화 | 미구현 (대안 채택) | pgvector 유사도 검색과 근본적으로 충돌 — 임베딩 컬럼 암호화 시 그래프 RAG 붕괴. 관리형 디스크 암호화를 README에 권장 |
| Edge TTS 교체 | 미구현 (문서화만) | 코드 변경보다 벤더 계약(Azure) 결정이 선행되어야 함. README에 출시 전 필수로 명시 |
| 만 14세 미만 연령 확인 | 미구현 | P1 스코프에 없었음. 일반 대중 출시 시 추가 필요 |
| 처리방침 플레이스홀더(`{{사업자명}}` 등) | 미완성 | 법적 문안이라 임의로 확정하지 않음. **변호사 검토 후 채울 것** |
| 프론트 실기기 런타임 검증 | 미실시 | `flutter analyze`만 통과. 실제 기기에서 동의 게이트·성문 토글 흐름 확인 필요 |
